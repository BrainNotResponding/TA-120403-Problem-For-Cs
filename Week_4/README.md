# Program Flowchart, Pseudo-code, and Memory Mapping Guide

โปรเจกต์นี้จัดทำขึ้นเพื่อเป็นคู่มือศึกษาเกี่ยวกับการจำลองการทำงานของโปรแกรมคอมพิวเตอร์ผ่าน **ผังงาน (Flowchart)**, **รหัสเทียม (Pseudo-code)**, พฤติกรรมการจัดเก็บค่าใน **หน่วยความจำ (Memory)** รวมถึงการทำความเข้าใจข้อผิดพลาดประเภทต่างๆ ที่มักเกิดขึ้นในกระบวนการเขียนโปรแกรม

## 🧠 2. กลไกการ Mapping ระหว่าง ผังงาน -> รหัสเทียม -> หน่วยความจำ

เมื่อเราเขียนโครงสร้างโปรแกรมเสร็จแล้ว คอมพิวเตอร์จะจำลองพื้นที่ใน **Memory (หน่วยความจำ)** เพื่อจองกล่องเก็บข้อมูลตามตัวแปรที่ระบุ โดยทำงานสัมพันธ์กันดังนี้:

### 🔹 ขั้นที่ 1: การประกาศและรับค่า (Read Phase)
*   **ผังงาน & รหัสเทียม:** สั่งคำสั่ง `read width, length` (เจอสัญลักษณ์สี่เหลี่ยมคางหมูรับค่าจากคีย์บอร์ด)
*   **หน่วยความจำทำงานยังไง:** คอมพิวเตอร์จะเปิดกล่องใน Memory ชื่อ `width` และ `length` รอไว้ พอมนุษย์พิมพ์ตัวเลขเข้ามา ค่าจะมุดเข้าไปจองในกล่องนั้นทันที

### 🔹 ขั้นที่ 2: การคำนวณ (Process Phase)
*   **ผังงาน & รหัสเทียม:** ทำงานที่บล็อก `area = width * length`
*   **หน่วยความจำทำงานยังไง:** คอมพิวเตอร์จะวิ่งไป "ส่องดูค่า" ในกล่อง `width` และกล่อง `length` นำมาคูณกันตามกระบวนการคิว จากนั้นจะนำผลลัพธ์ที่ได้ไป "โยนใส่" ในกล่องใหม่ที่ชื่อว่า `area`

### 🔹 ขั้นที่ 3: การส่งออก (Print Phase)
*   **ผังงาน & รหัสเทียม:** ทำงานที่บล็อก `print area`
*   **หน่วยความจำทำงานยังไง:** ดึงค่าที่สรุปเสร็จแล้วในกล่อง `area` จาก Memory ออกมาพ่นแสดงผลทางหน้าจอภาพคอมพิวเตอร์

## ⚠️ 3. การดักจับข้อผิดพลาดของโปรแกรม (Common Errors)

ในการเปลี่ยนผังงาน/รหัสเทียมให้กลายเป็นโค้ดโปรแกรมจริง (เช่น ภาษา C++) มักเกิด Error หลักๆ 3 รูปแบบที่เราต้องระวัง:

### 1️⃣ กฎการตั้งชื่อ (Naming Rules Error)
*   **ทำงานยังไง:** คอมพิวเตอร์ไม่เข้าใจชื่อตัวแปรหรือชื่อไฟล์ที่มีสัญลักษณ์แปลกปลอม
*   **ข้อห้าม:** ห้ามตั้งชื่อมีเว้นวรรค, ห้ามใช้อักขระพิเศษ (ยกเว้นเครื่องหมาย `_` Underscore ที่ใช้ได้), และห้ามใช้ภาษาไทยเด็ดขาด

### 2️⃣ Syntax Error (ข้อผิดพลาดทางไวยากรณ์)
*   **ทำงานยังไง:** เกิดจากการพิมพ์โค้ดผิดกฎเกณฑ์ของภาษานั้นๆ ทำให้ **Compiler ไม่ยอมผ่าน** และสั่งรันโปรแกรมไม่ได้
*   **สิ่งที่มักพลาด:** ลืมปิดท้ายด้วยเครื่องหมาย semicolon `;`, ลืมประกาศตัวแปรสะกดชื่อผิด, หรือสะกดคำสั่งพื้นฐานผิด เช่น พิมพ์ `prinf` (ขาด t) หรือ `return0` (พิมพ์ติดกัน)

### 3️⃣ Runtime Error (ข้อผิดพลาดขณะรันโปรแกรม)
*   **ทำงานยังไง:** โค้ดผ่านการตรวจไวยากรณ์เรียบร้อย (Compile ผ่านไร้ปัญหา) แต่พอเปิดโปรแกรมทำงานจริง โปรแกรมกลับทำงานไม่สำเร็จหรือเกิดอาการค้าง (Crash)
*   **ตัวอย่างที่พบบ่อย:** **การรัน Loop ไม่รู้จบ (Infinite Loop)** ซึ่งทำให้หน่วยความจำและ CPU ทำงานหนักจนโปรแกรมค้างไปเอง
