# Program Flowchart, Pseudo-code, and Memory Mapping Guide

โปรเจกต์นี้จัดทำขึ้นเพื่อเป็นคู่มือศึกษาเกี่ยวกับการจำลองการทำงานของโปรแกรมคอมพิวเตอร์ผ่าน ผังงาน (Flowchart), รหัสเทียม (Pseudo-code), พฤติกรรมการจัดเก็บค่าใน หน่วยความจำ (Memory) รวมถึงการทำความเข้าใจข้อผิดพลาดประเภทต่างๆ ที่มักเกิดขึ้นในกระบวนการเขียนโปรแกรม

---

## 1. ความหมายของสัญลักษณ์ผังงาน (Flowchart Symbols)

สัญลักษณ์เหล่านี้ใช้ในการวางแผนการไหลของข้อมูล (Data Flow) จากซ้ายไปขวาและบนลงล่าง เพื่อควบคุมลำดับคำสั่งให้คอมพิวเตอร์ทำงานตามขั้นตอนได้อย่างถูกต้อง[cite: 1]

| สัญลักษณ์ผังงาน (ASCII Shape) | ชื่อเรียก (Name) | การทำงาน / สิ่งที่ต้องระบุภายใน | ทำเพื่ออะไร (Purpose) |
| :--- | :--- | :--- | :--- |
| [ Start / Stop ] | Start / Stop | จุดเริ่มต้นและสิ้นสุดโปรแกรม ต้องระบุคำว่า "start" หรือ "stop" เสมอ[cite: 1] | เพื่อเป็นจุดเริ่มต้นและจุดสิ้นสุดของอัลกอริทึม[cite: 1] |
| +---------------+<br>\|    Process    \|<br>+---------------+ | Process | คำนวณหรือประมวลผลข้อมูล เช่น "area = width * length" หรือกำหนดค่าตัวแปร[cite: 1] | เพื่อสั่งให้ระบบทำการคำนวณ เปลี่ยนแปลงค่า หรือโยกย้ายข้อมูลในระบบ[cite: 1] |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;/\<br>&nbsp;&nbsp;&nbsp;&nbsp;/&nbsp;&nbsp;&nbsp;&nbsp;\\<br>&nbsp;&nbsp;/Decision\\<br>&nbsp;&nbsp;\&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;/<br>&nbsp;&nbsp;&nbsp;&nbsp;\&nbsp;&nbsp;&nbsp;&nbsp;/<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;\\/ | Decision | การตัดสินใจทางตรรกะ เช่น "x > 10" เพื่อแยกการทำงานออกเป็นทาง True หรือ False[cite: 1] | เพื่อสร้างเงื่อนไขและแยกเส้นทางการทำงานของโปรแกรม[cite: 1] |
| &nbsp;&nbsp;&nbsp;/---------------/<br>&nbsp;&nbsp;/ Input / Output /<br>&nbsp;/---------------/ | Input / Output | รับข้อมูลหรือแสดงผลข้อมูลทั่วไป โดยไม่ได้เจาะจงชนิดของอุปกรณ์[cite: 1] | เป็นคำสั่งรับเข้าหรือส่งออกข้อมูลในภาพรวม[cite: 1] |
| &nbsp;&nbsp;&nbsp;/---------------/<br>&nbsp;&nbsp;/ Read Keyboard /<br>&nbsp;/_______________/ | Input from Keyboard | รับข้อมูลผ่านทางคีย์บอร์ด ต้องระบุชื่อตัวแปรที่จะนำค่าไปเก็บ เช่น "read width"[cite: 1] | เพื่อรับค่าใหม่ๆ จากผู้ใช้ผ่านคีย์บอร์ดเข้ามาประมวลผลในระบบ[cite: 1] |
| +---------------+\<br>\| Print Monitor  ><br>+---------------+ | Output to Monitor | แสดงผลลัพธ์ออกทางจอภาพ ต้องระบุตัวแปรหรือข้อความ เช่น "print area"[cite: 1] | เพื่อสื่อสารผลลัพธ์หรือข้อมูลล่าสุดให้ผู้ใช้งานมองเห็นบนหน้าจอ[cite: 1] |
| +-------+<br>\| Database \|<br>+-------+ | Magnetic Disk / Database | จัดเก็บหรือดึงข้อมูลจากระบบฐานข้อมูล[cite: 1] | เพื่อบันทึกข้อมูลแบบถาวรลงในดิสก์[cite: 1] |

## 2. กลไกการเชื่อมโยงข้อมูล (Flowchart -> Pseudo-code -> Memory)

เมื่อเราแปลงผังงานและรหัสเทียมไปเป็นโค้ดโปรแกรม คอมพิวเตอร์จะจำลองพื้นที่จัดเก็บข้อมูลในหน่วยความจำ (Memory) เป็น "กล่องรับข้อมูล" ตามรายชื่อตัวแปรที่ระบุไว้ โดยทำงานร่วมกันเป็นขั้นตอนดังนี้:[cite: 1]

### ขั้นที่ 1: การรับค่าเข้าสู่ระบบ (Read Phase)
* **พฤติกรรมคำสั่ง:** เมื่อเจอคำสั่ง "read width, length"[cite: 1]
* **การทำงานของคอมพิวเตอร์:** คอมพิวเตอร์จะเปิดพื้นที่กล่องตัวแปรชื่อ "width" และ "length" ใน Memory รอไว้[cite: 1] เมื่อผู้ใช้พิมพ์ตัวเลขเข้ามา ค่าเหล่านั้นจะถูกส่งเข้าไปจองในกล่องทันที[cite: 1]

### ขั้นที่ 2: การประมวลผลคำนวณ (Process Phase)
* **พฤติกรรมคำสั่ง:** เมื่อเจอคำสั่ง "area = width * length"[cite: 1]
* **การทำงานของคอมพิวเตอร์:** คอมพิวเตอร์จะดึงข้อมูลที่อยู่ในกล่อง "width" และ "length" จาก Memory มาคูณกัน[cite: 1] จากนั้นนำผลลัพธ์สุดท้ายไปเขียนทับลงในกล่องใหม่ที่ชื่อว่า "area"[cite: 1]

### ขั้นที่ 3: การส่งออกผลลัพธ์ (Print Phase)
* **พฤติกรรมคำสั่ง:** เมื่อเจอคำสั่ง "print area"[cite: 1]
* **การทำงานของคอมพิวเตอร์:** คอมพิวเตอร์จะวิ่งไปอ่านค่าล่าสุดที่จัดเก็บอยู่ในกล่อง "area" ใน Memory เพื่อนำออกมาพ่นแสดงผลทางจอภาพ[cite: 1]

## 3. รูปแบบข้อผิดพลาดในการเขียนโปรแกรม (Common Errors)

เมื่อนำโครงสร้างอัลกอริทึมไปเขียนเป็นโค้ดจริง (เช่น ภาษา C/C++) มักพบข้อผิดพลาดหลักๆ อยู่ 3 รูปแบบ:[cite: 1]

### 1. Error จากการตั้งชื่อ (Naming Rules Error)
* **สิ่งที่เกิดขึ้น:** คอมพิวเตอร์ประมวลผลโค้ดไม่ได้เนื่องจากชื่อไฟล์หรือชื่อตัวแปรขัดต่อหลักการ[cite: 1]
* **สิ่งที่ต้องระวัง:** ห้ามตั้งชื่อตัวแปรมีช่องว่างเว้นวรรค, ห้ามใช้อักขระพิเศษ (ยกเว้นเครื่องหมาย _ Underscore), และห้ามใช้ภาษาไทย[cite: 1]

### 2. Syntax Error (ข้อผิดพลาดทางไวยากรณ์)
* **สิ่งที่เกิดขึ้น:** เกิดจากการเขียนโค้ดผิดกฎระเบียบทางภาษา ทำให้ Compiler ไม่ยอมผ่านและไม่สร้างไฟล์โปรแกรมเพื่อรันทำงาน[cite: 1]
* **สิ่งที่มักพลาด:**
  * การลืมใส่เครื่องหมาย Semicolon (;) ปิดท้ายคำสั่ง[cite: 1]
  * ลืมประกาศชื่อตัวแปรแต่ดึงไปใช้งานเลย[cite: 1]
  * สะกดคำสั่งพื้นฐานผิดพลาด เช่น พิมพ์ "prinf" แทน "printf" หรือพิมพ์ "return0" ติดกัน[cite: 1]

### 3. Runtime Error (ข้อผิดพลาดขณะโปรแกรมทำงาน)
* **สิ่งที่เกิดขึ้น:** โค้ดผ่านการตรวจสอบไวยากรณ์เรียบร้อยและเปิดโปรแกรมทำงานได้ปกติ แต่โปรแกรมจะหยุดทำงานกระทันหัน (Crash) หรือทำงานไม่สำเร็จเมื่อมีเงื่อนไขบางอย่างเกิดขึ้นในระหว่างการทำงานจริง[cite: 1]
* **สิ่งที่มักพลาด:** การเขียนลูปประมวลผลที่ไม่มีเงื่อนไขหลุดออก (Infinite Loop / การรันลูปไม่รู้จบ) ซึ่งส่งผลให้ CPU ทำงานหนักจนระบบค้างไปเอง[cite: 1]
