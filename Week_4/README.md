# Program Flowchart, Pseudo-code, and Memory Mapping Guide

คู่มืออธิบายการทำงานของระบบคอมพิวเตอร์ผ่านผังงาน (Flowchart) รหัสเทียม (Pseudo-code) การจำลองพื้นที่ในหน่วยความจำ (Memory Map) และการวิเคราะห์ข้อผิดพลาดพื้นฐานในการเขียนโปรแกรม

---

## 1. คำอธิบายและการใช้งานสัญลักษณ์ผังงาน (Flowchart Symbols)

ตารางสรุปสัญลักษณ์มาตรฐานที่ใช้ในการออกแบบกระบวนการทำงานของระบบคอมพิวเตอร์

| สัญลักษณ์ (Symbol) | ชื่อเรียก (Name) | การทำงาน / สิ่งที่ต้องระบุภายใน | ทำเพื่ออะไร (Purpose) |
| :--- | :--- | :--- | :--- |
| **วงรี (Oval)** | Start / Stop | จุดเริ่มต้นและจุดสิ้นสุดของโปรแกรม โดยระบุคำว่า "start" หรือ "stop" ไว้ภายใน | เพื่อแสดงจุดเริ่มต้นและจุดสิ้นสุดของกระบวนการทำงานทั้งหมด |
| **สี่เหลี่ยมผืนผ้า (Rectangle)** | Process | การประมวลผล เช่น การคำนวณสูตรคณิตศาสตร์ หรือการกำหนดค่าตัวแปร (เช่น area = width * length) | เพื่อสั่งให้ระบบดำเนินการคำนวณหรือเปลี่ยนแปลงค่าข้อมูลภายใน |
| **สี่เหลี่ยมข้าวหลามตัด (Diamond)** | Decision | การตัดสินใจทางตรรกะ โดยระบุเงื่อนไขเปรียบเทียบ เช่น "x > 10" เพื่อแยกเส้นทางเป็น True หรือ False | เพื่อสร้างทางเลือกในการตัดสินใจ และกำหนดเส้นทางการประมวลผลขั้นถัดไป |
| **สี่เหลี่ยมด้านขนาน (Parallelogram)** | Input / Output | การนำข้อมูลเข้าหรือแสดงผลข้อมูลทั่วไป โดยไม่ได้เจาะจงชนิดของอุปกรณ์ | เป็นคำสั่งในการรับหรือส่งออกข้อมูลในระดับทั่วไป |
| **สี่เหลี่ยมคางหมูเอียง (Trapezoid)** | Input from Keyboard | รอรับข้อมูลจากผู้ใช้งานผ่านคีย์บอร์ด โดยระบุชื่อตัวแปรที่ต้องการนำไปเก็บค่า (เช่น read width) | เพื่อให้โปรแกรมสามารถรับค่าที่ยืดหยุ่นจากผู้ใช้งานขณะรันโปรแกรมได้ |
| **บล็อกปลายแหลม (Display)** | Output to Monitor | การแสดงผลลัพธ์ออกทางหน้าจอภาพ โดยระบุชื่อตัวแปรหรือข้อความที่ต้องการแสดงผล (เช่น print area) | เพื่อสื่อสารข้อมูล ผลลัพธ์ หรือข้อความแจ้งเตือนให้ผู้ใช้งานได้รับทราบ |
| **ทรงกระบอก (Database)** | Database / Magnetic Disk | การอ่านหรือเขียนข้อมูลลงในระบบฐานข้อมูลหรือดิสก์จัดเก็บข้อมูล | เพื่อบันทึกข้อมูลไว้ใช้งานอย่างถาวร (ไม่สูญหายเมื่อปิดโปรแกรม) |
| **วงกลมเล็ก (Circle)** | Connector | จุดเชื่อมต่อย่อยภายในหน้ากระดาษเดียวกัน | เพื่อเชื่อมโยงเส้นทางการไหลของข้อมูลที่มีความซับซ้อนให้เป็นระเบียบ |

## 2. ขั้นตอนการประมวลผลข้อมูล (Flowchart -> Pseudo-code -> Memory)

เมื่อโปรแกรมทำงานตามผังงาน คอมพิวเตอร์จะประสานงานร่วมกับหน่วยความจำหลัก (Memory) เพื่อจำลองพื้นที่จัดเก็บข้อมูลตามรายชื่อตัวแปร โดยทำงานเรียงตามลำดับดังนี้:

### ขั้นตอนที่ 1: การรับค่า (Read)
* **การทำงาน:** คอมพิวเตอร์ทำงานผ่านคำสั่งรับค่า เช่น `read width, length`
* **พฤติกรรมในหน่วยความจำ:** ระบบจะทำการสร้างพื้นที่จองเก็บข้อมูล (กล่องตัวแปร) ชื่อ `width` และ `length` ใน Memory เพื่อสแตนด์บายรอรับตัวเลขที่ผู้ใช้จะพิมพ์เข้ามา

### ขั้นตอนที่ 2: การคำนวณ (Process)
* **การทำงาน:** คอมพิวเตอร์ประมวลผลสูตรคำนวณ เช่น `area = width * length`
* **พฤติกรรมในหน่วยความจำ:** ระบบจะวิ่งไปดึงค่าตัวเลขจากกล่อง `width` และ `length` ใน Memory ออกมาคำนวณคูณกัน จากนั้นจะส่งผลลัพธ์ที่ได้ไปบันทึกเก็บไว้ในกล่องที่ชื่อว่า `area`

### ขั้นตอนที่ 3: การแสดงผล (Print)
* **การทำงาน:** คอมพิวเตอร์ทำงานผ่านคำสั่งแสดงผลลัพธ์ เช่น `print area`
* **พฤติกรรมในหน่วยความจำ:** ระบบจะเปิดอ่านค่าล่าสุดที่ถูกจัดเก็บอยู่ในกล่อง `area` แล้วส่งค่านั้นออกมาแสดงผลบนหน้าจอภาพของผู้ใช้งาน

## 3. ประเภทข้อผิดพลาดที่พบบ่อยในการเขียนโปรแกรม (Common Errors)

เมื่อนำการออกแบบผังงานไปพัฒนาเป็นซอร์สโค้ดจริง (เช่น ภาษา C/C++) มักเกิดข้อผิดพลาดที่ต้องระวัง 3 รูปแบบหลัก:

### 1. Error จากการตั้งชื่อ (Naming Rules Error)
* **สาเหตุการเกิด:** ตัวแปลภาษาไม่เข้าใจชื่อตัวแปรหรือชื่อไฟล์ที่ตั้งขึ้นขัดต่อกฎ
* **แนวทางแก้ไข:** หลีกเลี่ยงการเว้นวรรคในชื่อตัวแปร, ห้ามใช้อักขระพิเศษ (ยกเว้นเครื่องหมาย Under Score `_`), และห้ามใช้ภาษาไทยในการเขียนโค้ดและตั้งชื่อไฟล์

### 2. Syntax Error (ข้อผิดพลาดทางไวยากรณ์)
* **สาเหตุการเกิด:** เขียนโค้ดผิดกฎระเบียบของภาษานั้นๆ ส่งผลให้ **Compiler ไม่ผ่าน** และไม่สามารถสร้างโปรแกรมสำหรับรันได้
* **ตัวอย่างที่พบบ่อย:** การลืมปิดท้ายคำสั่งด้วย Semicolon (`;`), สะกดคำสั่งพื้นฐานผิด (เช่น พิมพ์ `prinf` แทน `printf` หรือพิมพ์ `return0` ติดกัน), และการเรียกใช้ตัวแปรที่ไม่ได้ประกาศไว้ก่อนหน้า

### 3. Runtime Error (ข้อผิดพลาดขณะโปรแกรมทำงาน)
* **สาเหตุการเกิด:** โค้ดผ่านการตรวจสอบไวยากรณ์และ Compile ผ่านปกติ แต่เกิดการทำงานที่ผิดพลาดจนโปรแกรมหยุดทำงานกะทันหัน (Crash) ระหว่างรันจริง
* **ตัวอย่างที่พบบ่อย:** การเขียนลูปทำงานที่ไม่มีเงื่อนไขหยุดออก (Infinite Loop) ส่งผลให้ตัวประมวลผล (CPU) และหน่วยความจำทำงานหนักจนโปรแกรมค้าง
