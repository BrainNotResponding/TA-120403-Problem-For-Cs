# Operator Precedence & Expression Evaluation Tracker

โปรเจกต์นี้จัดทำขึ้นเพื่อศึกษาและวิเคราะห์ **ลำดับความสำคัญของตัวดำเนินการ (Operator Precedence และ Associativity)** ทั้งในรูปแบบของนิพจน์คณิตศาสตร์ (Arithmetic Expressions) และนิพจน์ตรรกศาสตร์ (Logical Expressions) เพื่อให้เข้าใจขั้นตอนการประมวลผลภายในระบบคอมพิวเตอร์อย่างถูกต้อง

---

## 📖 เนื้อหาและหลักการทำงาน (How it works)

ระบบคอมพิวเตอร์มีหลักเกณฑ์ในการคำนวณนิพจน์ที่ซับซ้อน โดยเรียงลำดับจากเครื่องหมายที่มีความสำคัญมากที่สุดไปยังน้อยที่สุด หากเครื่องหมายมีความสำคัญเท่ากัน จะพิจารณาทิศทางการทำงาน (จากซ้ายไปขวา หรือ ขวาไปซ้าย) 

### 1. ตารางลำดับความสำคัญของเครื่องหมาย (Operator Precedence Table)

| ลำดับความสำคัญ | เครื่องหมาย | ประเภท Operator | ชนิด | การทำงานพิจารณาจาก |
| :---: | :---: | :---: | :---: | :---: |
| **1 (มากสุด)** | `()` | วงเล็บ | - | ซ้ายไปขวา |
| **2** | `+`, `-` | Arithmetic | Unary | **ขวาไปซ้าย** |
| **3** | `!` (`~`, `not`) | Logical | Unary | - |
| **4** | `*`, `/`, `%` | Arithmetic | Binary | ซ้ายไปขวา |
| **5** | `+`, `-` | Arithmetic | Binary | ซ้ายไปขวา |
| **6** | `<`, `<=`, `>`, `>=` | Relational | Binary | ซ้ายไปขวา |
| **7** | `==`, `!=` | Relational | Binary | ซ้ายไปขวา |
| **8** | `&&` (`and`) | Logical | Binary | ซ้ายไปขวา |
| **9** | `\|\|` (`or`) | Logical | Binary | ซ้ายไปขวา |
| **10 (น้อยสุด)** | `=`, `+=`, `-=`, `*=`, `/=` | Assignment | - | **ขวาไปซ้าย** |

## 🛠 ส่วนประกอบและการทำงานของระบบ (System Structure)

โปรเจกต์นี้จะแปลงนิพจน์ทั่วไปให้อยู่ในรูปแบบ **ลำดับการประมวลผลแบบคอมพิวเตอร์ (Computer Evaluation Tree)** โดยแบ่งการทดสอบออกเป็น 3 ส่วนหลัก:

### 🔹 ส่วนที่ 1: แปลงนิพจน์คณิตศาสตร์พื้นฐาน (Basic Arithmetic Rules)
ทำหน้าที่จัดกลุ่มและใส่วงเล็บให้แก่นิพจน์ เพื่อบังคับให้คอมพิวเตอร์คำนวณตามลำดับ Precedence ที่ถูกต้อง
*   **ตัวอย่างการทำงาน:**
    *   โจทย์: `5 - 2 + 6 / 2 * 3`
    *   รูปวงเล็บที่ถูกต้อง: `(5 - 2) + ((6 / 2) * 3)`
    *   เหตุผล: เครื่องหมาย `/` และ `*` สำคัญกว่า `+` และ `-` จึงต้องคิดกลุ่มคูณหารก่อน โดยไล่จากซ้ายไปขวา

### 🔹 ส่วนที่ 2: วิเคราะห์โครงสร้างต้นไม้คณิตศาสตร์แบบซับซ้อน (Complex Expression Tree)
วิเคราะห์และประมวลผลโจทย์คณิตศาสตร์ที่มีทั้วแปรและวงเล็บซ้อนวงเล็บ โดยแสดงผลลัพธ์ทีละ Step
*   **ตัวอย่างสมการ:** `5 * Y + (3 + Z - 1) / (4 * (3 * Z + 1) - Y)`
*   **วิธีการทำงาน:** ระบบจะเจาะลึกเข้าไปคำนวณในวงเล็บในสุดก่อน (`3 * Z + 1`) แล้วค่อยๆ ขยับออกมาคำนวณด้านนอกตามลำดับเส้นเชื่อม (Evaluation Tree) จนได้ผลลัพธ์สุดท้าย

### 🔹 ส่วนที่ 3: นิพจน์เปรียบเทียบและตรรกศาสตร์ (Relational & Logical Evaluation)
ประมวลผลเงื่อนไขที่ให้ผลลัพธ์เป็น `true` หรือ `false` โดยผสมผสานเครื่องหมายคณิตศาสตร์, เครื่องหมายเปรียบเทียบ (`==`, `<=`) และเครื่องหมายตรรกะ (`&&`, `||`, `!`)
*   **ตัวอย่างการประมวลผล:** `x || z == !y && !x`
*   **การใส่กลุ่มวงเล็บที่คอมพิวเตอร์ทำงาน:** `x || ((z == (!y)) && (!x))`
*   **หลักการประมวลผลลัพธ์:** 
    1. ทำ Unary Operator ก่อน เช่น `!y` และ `!x`
    2. ทำเครื่องหมายเปรียบเทียบ `==`
    3. ทำเครื่องหมาย `&&` (และ)
    4. ทำเครื่องหมาย `||` (หรือ) เป็นลำดับสุดท้าย

## 🚀 วิธีการใช้งาน / สิ่งที่ต้องใส่ (Inputs & Specifications)

หากต้องการนำไปเขียนโค้ดต่อหรือใช้งานโปรแกรม ตัวแปรที่ต้องกำหนดค่า (Input) ประกอบด้วย:

1.  **ค่าสำหรับนิพจน์คณิตศาสตร์ (Numeric Inputs):** กำหนดค่าตัวแปร เช่น `Y = 1`, `Z = 3` (ตามโจทย์ตัวอย่าง) เพื่อใช้คำนวณค่าตัวเลข
2.  **ค่าสำหรับนิพจน์ตรรกศาสตร์ (Boolean Inputs):** กำหนดสถานะ `true` หรือ `false` ให้แก่ตัวแปร เช่น `x = true`, `y = false`, `z = false` รวมถึงตัวแปร `a`, `b`, `c`

### ตัวอย่างการรันผลลัพธ์ (Sample Output)
เมื่อป้อนนิพจน์ตรรกศาสตร์ที่ซับซ้อนเข้าไป ระบบจะทำการแจกแจงโครงสร้างต้นไม้ (Tree Tree Step) ออกมาเป็นค่าความจริงในแต่ละชั้น จนได้คำตอบสุทธิ เช่น `ผลลัพธ์สุดท้าย = true`
