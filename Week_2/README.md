# Operator Precedence & Expression Evaluation Tracker

โปรเจกต์นี้จัดทำขึ้นเพื่อศึกษาและวิเคราะห์ **ลำดับความสำคัญของตัวดำเนินการ (Operator Precedence และ Associativity)** ทั้งในรูปแบบของนิพจน์คณิตศาสตร์ (Arithmetic Expressions) และนิพจน์ตรรกศาสตร์ (Logical Expressions) เพื่อให้เข้าใจขั้นตอนการประมวลผลภายในระบบคอมพิวเตอร์อย่างถูกต้อง

---

## 📖 เนื้อหาและหลักการทำงาน (How it works)

ระบบคอมพิวเตอร์มีหลักเกณฑ์ในการคำนวณนิพจน์ที่ซับซ้อน โดยเรียงลำดับจากเครื่องหมายที่มีความสำคัญมากที่สุดไปยังน้อยที่สุด หากเครื่องหมายมีความสำคัญเท่ากัน จะพิจารณาทิศทางการทำงาน (จากซ้ายไปขวา หรือ ขวาไปซ้าย) 

### 1. ตารางลำดับความสำคัญของเครื่องหมาย (Operator Precedence Table)

| ลำดับความสำคัญ | เครื่องหมาย | ประเภท Operator | ชนิด | การทำงานพิจารณาจาก |
| :---: | :---: | :---: | :---: | :---: |
| **1 (มากสุด)** | `()` | วงเล็บ | - | ซ้ายไปขวา |
| **2** | `+`, `-` | Arithmetic | Unary | **ขวาไปซ้าย** |
| **3** | `!` (`~`, `not`) | Logical | Unary | - |
| **4** | `*`, `/`, `%` | Arithmetic | Binary | ซ้ายไปขวา |
| **5** | `+`, `-` | Arithmetic | Binary | ซ้ายไปขวา |
| **6** | `<`, `<=`, `>`, `>=` | Relational | Binary | ซ้ายไปขวา |
| **7** | `==`, `!=` | Relational | Binary | ซ้ายไปขวา |
| **8** | `&&` (`and`) | Logical | Binary | ซ้ายไปขวา |
| **9** | `\|\|` (`or`) | Logical | Binary | ซ้ายไปขวา |
| **10 (น้อยสุด)** | `=`, `+=`, `-=`, `*=`, `/=` | Assignment | - | **ขวาไปซ้าย** |

## 🛠 ส่วนประกอบและการทำงานของระบบ (System Structure)

โปรเจกต์นี้จะแปลงนิพจน์ทั่วไปให้อยู่ในรูปแบบ **ลำดับการประมวลผลแบบคอมพิวเตอร์ (Computer Evaluation Tree)** โดยแบ่งการทดสอบออกเป็น 3 ส่วนหลัก:


### 🔹 ส่วนที่ 1: การวิเคราะห์นิพจน์คณิตศาสตร์พื้นฐาน (Basic Arithmetic)
* **การทำงาน:** สแกนจากซ้ายไปขวาเพื่อใส่วงเล็บจัดกลุ่มความสำคัญ (เช่น คูณ/หาร ต้องถูกเคลียร์ก่อน บวก/ลบ)
* **ตัวอย่าง:** `5 - 2 + 6 / 2 * 3` 
* **ผลลัพธ์การจัดกลุ่ม:** `(5 - 2) + ((6 / 2) * 3)` คอมพิวเตอร์จะเคลียร์วงเล็บก้อนซ้าย `(5 - 2)` ได้ `3` จากนั้นเดินหน้าไปทำก้อนขวา มุดเข้าทำ `6 / 2` ได้ `3` แล้วนำไปคูณ `3` ได้ `9` ก่อนนำผลลัพธ์มารวมกัน

### 🔹 ส่วนที่ 2: การคำนวณนิพจน์คณิตศาสตร์แบบซับซ้อน (Complex Expression Tree)
* **โจทย์ทดสอบ:** `5 * Y + (3 + Z - 1) / (4 * (3 * Z + 1) - Y)`
* **ลำดับการมุดของคอมพิวเตอร์:** 
  1. สแกนเจอและมุดเข้าทำวงเล็บก้อนแรก `(3 + Z - 1)` ทางซ้ายจนเสร็จ
  2. เดินหน้าต่อไปฝั่งขวา มุดเข้าวงเล็บใหญ่ `(4 * ... - Y)`
  3. ขณะสแกนในวงเล็บใหญ่ คิวเดินไปเจอวงเล็บซ้อนด้านใน จึงมุดลึกต่อเข้าไปทำ `(3 * Z + 1)` ให้เสร็จก่อน แล้วค่อยถอยมาทำวงเล็บชั้นนอก

### 🔹 ส่วนที่ 3: การประเมินค่านิพจน์ตรรกศาสตร์ (Relational & Logical Evaluation)
* **โจทย์ทดสอบ:** `x || z == !y && !x` และเงื่อนไขสายยาวในระบบใบงาน
* **การทำงาน:** คอมพิวเตอร์จะสแกนเพื่อเปลี่ยนค่าตัวแปร เช่น `x = true`, `y = false`, `z = false` จากนั้นประมวลผล Unary (`!`) ก่อน แล้วตามด้วยเครื่องหมายเปรียบเทียบ (`==`, `<=`) และทำการยุบค่าตามตรรกะ `&&` กับ `||` จากซ้ายไปขวา

## 🚀 สิ่งที่ต้องกำหนดให้ระบบ (Inputs)

เพื่อให้โปรแกรมคำนวณและสร้างแผนภาพต้นไม้ได้อย่างถูกต้อง ผู้ใช้ต้องกำหนดค่าดังนี้:
1. **ตัวแปรตัวเลข (Numeric Inputs):** กำหนดค่าให้ตัวแปรต้น เช่น `Y`, `Z` เพื่อประมวลผลสมการคณิตศาสตร์
2. **ตัวแปรตรรกะ (Boolean Inputs):** กำหนดค่า `true`/`false` ให้แก่ตัวแปรเงื่อนไข เช่น `x`, `y`, `z`, `a`, `b`, `c`

## 🎯 วัตถุประสงค์ของโปรเจกต์ (Objectives)
* เพื่อจำลองและแสดงผลลัพธ์การทำงานของคิวมนตรี (Execution Queue) ของคอมพิวเตอร์ได้อย่างถูกต้อง
* ช่วยให้ผู้พัฒนาซอฟต์แวร์เข้าใจพฤติกรรมการ Parse โค้ด ป้องกันข้อผิดพลาดในการเขียนเงื่อนไขคณิตศาสตร์และตรรกะที่ซับซ้อน
